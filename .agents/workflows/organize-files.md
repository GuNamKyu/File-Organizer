# 📂 파일 정리 워크플로우 (organize-files)

사용자가 `[경로 : 위치경로]` 형식으로 경로를 제공하면, 해당 경로의 파일을 스캔하고 확장자 기반으로 분류하여 넘버링된 폴더로 정리한다.

---

## 사전 준비

// turbo-all

이 워크플로우를 실행하기 전에 다음 설정 파일을 반드시 로드한다:

1. `config/rules.json` — 분류 규칙 (최대 깊이, 넘버링 형식, 기본 카테고리)
2. `config/extensions.json` — 확장자 → 카테고리 매핑 정보

---

## 실행 단계

### 1단계: 입력 파싱 및 경로 추출

사용자 메시지에서 `[경로 : ...]` 패턴을 찾아 대상 경로를 추출한다.

- 형식: `[경로 : C:\Users\example\path]`
- 경로에서 앞뒤 공백을 제거한다.
- 경로가 추출되지 않으면 사용자에게 올바른 형식을 안내하고 중단한다.

### 2단계: 경로 유효성 검증

`list_dir` 도구를 사용하여 대상 경로가 존재하고 접근 가능한지 확인한다.

```
list_dir(DirectoryPath: 추출된_경로)
```

- 경로가 존재하지 않거나 접근 불가능하면 사용자에게 알리고 중단한다.
- 경로가 유효하면 다음 단계로 진행한다.

### 3단계: 파일/폴더 목록 스캔

대상 경로의 **루트에 있는 파일들만** 스캔한다. (이미 분류된 하위 폴더 내 파일은 건드리지 않음)

스캔 결과를 다음과 같이 정리한다:

| 항목 | 설명 |
|------|------|
| 총 파일 수 | 루트에 있는 파일 개수 |
| 총 폴더 수 | 기존에 존재하는 폴더 개수 |
| 확장자별 분포 | 각 확장자의 파일 수 |

### 4단계: 설정 파일 로드

프로젝트 루트의 설정 파일을 읽는다:

```
view_file(config/rules.json)
view_file(config/extensions.json)
```

`rules.json`에서 다음 값을 확인한다:
- `maxDepth`: 최대 폴더 깊이 (기본값: 5)
- `folderNaming`: 폴더 넘버링 형식 (기본값: `{number}_{name}`)
- `numberFormat`: 번호 형식 (기본값: `"00"` → 01, 02, ...)
- `defaultCategory`: 매핑되지 않은 확장자의 기본 카테고리 (기본값: `"99_기타"`)
- `fileNaming`: 파일명 처리 방식 (항상 `"keep_original"` — 파일명 변경 없음)

### 5단계: 파일 분류 판단

각 파일의 확장자를 `extensions.json`의 카테고리와 매칭한다.

**분류 규칙:**
- 파일 확장자(소문자 변환 후)를 `extensions.json`의 각 카테고리에서 검색
- 매칭되는 카테고리가 있으면 해당 카테고리로 분류
- 매칭되지 않으면 `rules.json`의 `defaultCategory`(기본: `99_기타`)로 분류
- **파일명은 절대 변경하지 않는다** — 원래 파일명 그대로 유지

### 6단계: 폴더 구조 계획

분류 결과를 바탕으로 대상 폴더 구조를 설계한다.

**폴더 넘버링 규칙:**
- 기존에 넘버링된 폴더가 있으면 그 번호 체계를 유지한다
- 새로 필요한 폴더는 기존 번호 다음 순서로 부여한다
- 번호는 `01`, `02`, ... 형식 (2자리 0-패딩)
- 형식: `{번호}_{카테고리명}` (예: `01_문서`, `02_이미지`)
- **폴더 깊이가 5단계를 초과하지 않도록 검증한다**

**기존 넘버링 폴더 감지:**
- 대상 경로에 이미 `XX_이름` 패턴(XX는 숫자)의 폴더가 있는지 확인
- 있다면 해당 카테고리와 번호를 기존 구조에 맞춘다
- 기존 폴더에 파일을 추가하는 경우 폴더를 새로 만들지 않는다

### 7단계: 정리 계획 출력 및 사용자 승인 요청

정리 계획을 사용자에게 보여주고 승인을 요청한다.

**출력 형식:**
```
📊 파일 정리 계획:
══════════════════════════════════════════
  📁 대상 경로: [경로]
  📄 정리 대상 파일: [N]개
══════════════════════════════════════════

  파일명                    → 이동 경로
──────────────────────────────────────────
  보고서.pdf                → 01_문서/보고서.pdf
  사진.jpg                  → 02_이미지/사진.jpg
  노래.mp3                  → 03_음악/노래.mp3
  setup.exe                 → 04_실행파일/setup.exe
──────────────────────────────────────────

  📁 신규 폴더 생성 목록:
  - 04_실행파일
──────────────────────────────────────────
  ※ 파일명은 변경되지 않습니다. 폴더만 번호가 부여됩니다.

계속 진행하시겠습니까?
```

- 사용자가 승인하면 8단계로 진행
- 사용자가 거부하면 중단하고 수정 사항을 확인한다

### 8단계: 실행 (폴더 생성 → 파일 이동)

승인된 계획에 따라 실행한다.

**실행 순서:**
1. **신규 폴더 생성**: 필요한 폴더를 먼저 모두 생성
   ```powershell
   New-Item -ItemType Directory -Path "대상경로\01_문서" -Force
   ```

2. **파일 이동**: 각 파일을 해당 카테고리 폴더로 이동
   ```powershell
   Move-Item -Path "대상경로\파일명" -Destination "대상경로\01_문서\파일명"
   ```

**주의사항:**
- 이동 대상에 동일한 파일명이 이미 존재하면, 사용자에게 알리고 처리 방법을 확인한다 (덮어쓰기 / 건너뛰기 / 이름변경)
- 이동 중 오류가 발생하면 해당 파일을 건너뛰고 로그에 기록한다
- **파일명은 절대 변경하지 않고 이동만 수행한다**

### 9단계: 결과 보고

정리 완료 후 결과를 요약하여 보고한다.

**출력 형식:**
```
✅ 파일 정리 완료!
══════════════════════════════════════════
  📁 대상 경로: [경로]
  📄 정리된 파일: [N]개
  📁 생성된 폴더: [M]개
  ⚠️ 건너뛴 파일: [K]개
══════════════════════════════════════════

  카테고리별 정리 현황:
──────────────────────────────────────────
  01_문서       : 3개 파일
  02_이미지     : 5개 파일
  03_음악       : 2개 파일
  99_기타       : 1개 파일
──────────────────────────────────────────
```

## 엣지 케이스 처리

| 상황 | 처리 방법 |
|------|----------|
| 대상 경로에 파일이 없음 | "정리할 파일이 없습니다" 안내 후 중단 |
| 동일 파일명 충돌 | 사용자에게 처리 방법 확인 (덮어쓰기/건너뛰기/이름변경) |
| 확장자 없는 파일 | `defaultCategory` (99_기타)로 분류 |
| 숨김 파일 (`.`으로 시작) | 정리 대상에서 제외 |
| 폴더 깊이 5단계 초과 | 5단계까지만 허용, 초과 시 경고 후 현재 위치 유지 |
| 이미 넘버링된 폴더 존재 | 기존 번호 체계를 유지하고 이어서 번호 부여 |
| 읽기 전용 파일 | 이동 시도 후 실패하면 건너뛰고 로그에 기록 |
